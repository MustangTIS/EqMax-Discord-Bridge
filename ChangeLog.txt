■ v7.1.0 (Integration & Stability Update)
「境界を超えた情報の、その先へ」 — 接続性の拡張と、視認性の最適化。

今回のアップデートでは、Matrixサーバーとの互換性向上と、全プラットフォームにおける通知の視認性を劇的に改善しました。

[Protocol] Matrix API v3 完全対応

従来のr0形式から最新のv3へ移行。最新のMatrixホームサーバー環境においても、エラーなく安定した通知送信が可能になりました。

[Visual] クロスプラットフォーム通知の視認性最適化

SlackおよびMatrix向けに「通知区切り線」の自動挿入処理を実装。画像が添付されないテキストベースの環境でも、情報が視覚的に分離され、過去の速報との判別が容易になりました。

[Quality] タイムスタンプの標準化

送信時の timestamp を全プラットフォームのメッセージ末尾に付与。速報発生時刻と送信時刻が即座に確認できるようになり、情報の信頼性を向上させました。

[Setup] GUI インストーラの堅牢化

Webhook設定の検証ロジックを強化。HTTPS以外のURL登録を制限し、設定後の不具合を未然に防ぐ予防線を実装しました。

右クリックからのクリップボード貼り付けに対応（トークン等の誤入力リスクを低減）。

### ■ v7.0.0

**「境界を超えて、全方位へ」 — マルチプラットフォーム通知の夜明け。**

今回のメジャーアップデートでは、Discord への橋渡しという枠組みを完全に突破し、Slack や Matrix といった多様な通知プロトコルを統合しました。

* **[Platform] マルチ通知エンジン「Guardian Hub」の始動**
* Discord, Slack, Matrix への同時通知に対応。設定ツール（GUI）を刷新し、最大5箇所までの送信先を個別に定義可能に。
※GUIの高さが変更されるためログ情報を画面上部に移動

* **[Identity] Webhook送信名の強制統合**
* 送信時のユーザー名を `EqMax EEW Bridge` に統一。
* **[Warning] アイコンの制限について**: Discord の Webhook 仕様上、プログラム側から送信アイコンを動的に変更することはできません。任意のアイコンを設定したい場合は、Discord 側で作成した Webhook に手動でアイコンを設定いただくようお願いいたします。


* **[Stability] エラー診断プロンプトの強化**
* メインルーチン側に高度なエラーハンドリングを導入。送信失敗や環境エラーが発生した際、コンソール上で「何が原因か」を即座に特定できるよう情報を拡充しました。

### ■ v6.0.0 (Emergency & Performance Update)

**「静かなる守護者（The Silent Guardian）」 — 通信の完全分離と、究極の安定性へ。**

* **[Network] Twitter(X)機能の完全デッドロック回避（DummyPost Mode）**
* EqMax内部の隠しフラグ `TwitterDummyPost=1` を強制適用。通信の直前で処理を離脱させることで、CloudflareによるIPブロックやブラウザのフリーズを100%根絶しました。


* **[Setup] 「ワンクリック・パッチ」の完成**
* 初期設定パッチャーを刷新。ボタン一つで「ダミー認証の生成」「通信遮断フラグの注入」「キャプチャ設定の最適化」を全自動で行います。専門知識なしで「Discord専用機」への変身が完了します。


* **[Stability] ALL GREEN の真実味**
* ネットワークトラブルの火種を物理的に消し去ったことで、ダッシュボードの ALL GREEN は文字通り「一切の不安がない状態」を意味するようになりました。



### ■ v5.6.0 (Previous Update)

**「一歩先の安心を、その手に。」 — 監視エンジンの刷新と「嘘をつかない」起動シーケンスの導入**

* **[Core] 真のメモリリーク対策：Private Bytes監視への移行**
    * 従来の物理メモリ（RSS）監視を卒業し、アプリが確保している全領域を示すPrivate Bytesの監視を採用しました。潜在的なメモリ肥大化を確実にキャッチして対処します。

    * 起動プロセスの Step 1/7 ～ 7/7 をリアルタイムに可視化し、Webhookの待機状態まで全プロセスを把握可能にしました。
* **[Visual] ひと目でわかる「ALL GREEN」ダッシュボード**
    * 起動プロセスの最後に、システムの健全性を一言で判定するインターフェースを導入しました。
* **[Safety] 現場で直せる「セーフモード診断」の実装**
    * 万が一、ボットが起動しなくなった時のための「自己診断・自動修復機能」を強化しました。
